# Directus-style resource events

Ark resource hooks use `filter` before a write and `action` after it, with targeted namespaced events such as `ark.messages.items.create`. Ark does not emit generic cross-resource hooks such as `items.create`; their ordering and global effects are deferred until a concrete use case justifies them. Filters are blocking and participate in the operation transaction. Actions are required by default, are awaited inside the transaction, and roll the operation back on failure. An Action may explicitly opt into Directus-style best-effort handling, in which case Ark dispatches it asynchronously only after commit and logs failures. Durable external effects go through the outbox, and controlled internal operations may explicitly suppress event emission to prevent recursion.
