# Remote development uses isolated cells outside the Ark runtime

Ark tenant development uses persistent remote Development Cells, each with its own Postgres database, Meilisearch indexes, S3 buckets, and deterministic tunnel hostname. Generic orchestration lives in development-only `ark-dev` and `ark-tunnels` scripts shipped by the Ark package; tenant policy remains in `ARK_DEV_*` environment values and local tunnel config, while remote `frps` remains generic infrastructure. Neither the Ark runtime nor stage/production startup interprets Development Cell configuration. This keeps tenant development consistent without making remote providers, tenant names, or tunnel management part of Ark core, while avoiding extra source clones and leaving a straightforward path to extract standalone packages later.
